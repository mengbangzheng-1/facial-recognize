## 1. 产品目标与背景

### 1.1 背景

面部表情是人类情感传达的重要媒介，自动化表情识别在情感计算、人机交互、心理健康监测、安防监控等领域具有广泛应用价值。现有工业级方案往往依赖重量级模型，难以在边缘设备或实时场景中高效部署。

本项目基于学术论文成果，结合知识蒸馏与注意力机制对 MobileNetV3-Small 进行改进，训练一个兼顾精度与效率的轻量级表情识别模型，并配套可交互的可视化桌面应用。

### 1.2 产品目标

| 目标维度 | 具体指标 |
|---------|---------|
| 识别精度 | FER-2013 测试集准确率 ≥ 74.38% |
| 模型轻量化 | 参数量 ≤ 5M |
| 推理效率 | 单帧 CPU 推理延迟 ≤ 6ms |
| 实时视频 | 支持 ≥ 25 FPS 实时流处理 |
| 用户体验 | 提供直观的桌面 GUI，三种输入模式全覆盖 |

### 1.3 目标用户

- **研究者**：高校/企业 AI 研究人员，需要可复现的基线模型和实验数据
- **部署者**：将模型集成至下游系统（如情感监测平台、教育系统）的工程师
- **演示者**：需要向非技术受众展示 AI 表情识别能力的产品/市场人员

## 3. 需求池（P0 / P1 / P2 分级）

| ID | 模块 | 需求描述 |

| P0-01 | 模型训练 | 基于 FER-2013 数据集，使用改进 MobileNetV3-Small 完成训练，测试集准确率 ≥ 74.38% |

## 4. 技术约束与非功能需求

### 4.1 性能约束

| 指标 | 要求 |
|-----|------|
| 模型参数量 | ≤ 5M |
| 单帧 CPU 推理延迟 | ≤ 6ms（PyTorch 或 ONNX Runtime） |
| 实时视频帧率 | ≥ 25 FPS（包含人脸检测 + 表情分类全流程） |
| 测试集准确率 | ≥ 74.38%（FER-2013 官方测试集） |

### 4.2 技术栈约束

| 层次 | 技术选型 |
|-----|---------|
| 深度学习框架 | PyTorch（≥ 1.12） |
| 教师模型 | ConvNeXt-Base（torchvision 预训练权重） |
| 学生模型 | 改进 MobileNetV3-Small |
| GUI 框架 | PyQt5 或 PySide6 |
| 人脸检测 | OpenCV DNN / MediaPipe Face Detection |
| 数据集 | FER-2013（Kaggle 版本，35,887 张，48×48 灰度图） |
| 运行平台 | Windows 10+ / Ubuntu 20.04+，Python 3.8+ |
| 可选加速 | ONNX Runtime（CPU/GPU），CUDA ≥ 11.3（可选） |

### 4.3 非功能需求

| 类别 | 要求 |
|-----|------|
| 可维护性 | 代码结构模块化（数据/模型/训练/推理/GUI 分层），关键函数有注释 |
| 可复现性 | 固定随机种子，提供完整的依赖列表（requirements.txt） |
| 兼容性 | GUI 在 1080p 及以上分辨率下无遮挡，支持 HiDPI 缩放 |
| 安全性 | 不收集、不上传用户摄像头数据，纯本地运行 |
| 错误处理 | 摄像头不可用、视频解码失败、模型文件缺失时给出友好提示而非崩溃 |
| 安装便捷性 | 提供 requirements.txt 及 README，pip install 后可直接运行 |

## 6. 数据流与界面原型描述（文字版）

### 6.1 数据流

#### 6.1.1 训练数据流

```
FER-2013 原始数据（CSV/目录）
    ↓ 数据加载器（Dataset + DataLoader）
    ↓ 数据增强管道（在线增强）
    ↓ 归一化（ImageNet 均值/方差）
    ↓ 批次数据（batch_size=64, 48×48×1 灰度→3通道复制）
    ↓
┌─────────────────────────────────────┐
│  教师模型（ConvNeXt-Base）           │
│  → soft_labels（温度τ=4 的软概率）   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  学生模型（改进 MobileNetV3-Small）  │
│  → logits（7类原始输出）             │
└─────────────────────────────────────┘
    ↓
混合损失计算：0.7 × Focal Loss(logits, hard_labels)
           + 0.3 × KL(softmax(logits/τ) ‖ softmax(teacher_logits/τ))
    ↓
AdamW 反向传播 + 余弦退火更新
    ↓
验证集评估（每 epoch）→ 保存最优 checkpoint
```

#### 6.1.2 推理数据流（GUI 摄像头模式）

```
摄像头帧（BGR, 原始分辨率）
    ↓ 人脸检测（OpenCV DNN / MediaPipe）
    ↓ 人脸区域裁剪 + Resize(48×48)
    ↓ 灰度转换 → 3通道复制 → 归一化
    ↓ 模型推理（学生模型 forward）
    ↓ Softmax → 7类概率向量
    ↓ 置信度阈值过滤
    ↓ 绘制人脸框 + 表情标签 + 置信度
    ↓ 更新 GUI 左侧画面 + 右侧统计面板
    ↓ 循环（下一帧）
```

#### 交互说明

- **输入模式切换**：选择不同模式后，左侧画面区域适应性更新（图片为静态展示，视频/摄像头为动态播放）
- **开始/暂停/停止**：开始后按钮变为暂停可用；暂停后画面定格，统计数据保留；停止后清空画面
- **置信度阈值滑块**：拖动时实时生效，低于阈值的检测框仅显示框体，不显示表情标签
- **截图**：保存当前左侧画面（含所有标注）为 PNG，自动命名为 `fer_capture_YYYYMMDD_HHMMSS.png`
- **切换模型**：弹出文件选择对话框，选择 `.pth` 或 `.onnx` 文件，加载成功后状态栏提示
- **历史记录**：每次点击"停止"或关闭检测后，自动记录本次会话摘要（时间、模式、帧数、各表情计数）
